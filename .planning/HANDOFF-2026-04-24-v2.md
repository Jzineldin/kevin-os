## Key architectural decisions recorded today

### 1. AWS Organization SCP blocks Lambda Function URL `AuthType=NONE`
- Account `239541130189` is under Organization `o-g7pp3kd04f` (root `051542583152`)
- The SCP silently returns 403 even when resource policy is `Principal: "*"`
- **Workaround:** API Gateway HTTP API in front — id `v1k7d48lbk`, endpoint `https://v1k7d48lbk.execute-api.eu-north-1.amazonaws.com/`
- Routes: `ANY /{proxy+}` → dashboard-api ; `GET/POST /relay/{proxy+}` → relay-proxy

### 2. Bearer auth replaces SigV4
- Both dashboard-api + relay-proxy Lambdas verify `Authorization: Bearer <KOS_DASHBOARD_BEARER_TOKEN>` header (constant-time compare)
- Same shared secret as Vercel middleware gates the `/login` page with — `kos/dashboard-bearer-token`
- Vercel client `apps/dashboard/src/lib/dashboard-api.ts` sends Bearer header instead of signing with aws4fetch

### 3. Capture schema widened for multi-channel
- `CaptureReceivedTextSchema.channel` is now `z.enum(['telegram', 'dashboard'])` with optional `sender`/`telegram` fields
- `TriageRoutedSchema` carries optional channel + sender + telegram
- `EntityMentionDetectedSchema.source` added `'dashboard-text'`
- voice-capture handler skips the output.push Telegram reply emit for dashboard captures (SSE handles the user-facing ack via migration 0009 agent_runs trigger)

### 4. LLM enum drift tolerance
- `services/voice-capture/src/agent.ts::VoiceCaptureOutputSchema` preprocesses `candidate_type` via alias map before zod validation: `Organization`/`Company`/`Organisation` → `Org`, etc.
- Haiku 4.5 occasionally ignores the prompt's "Org" example and returns the long-form. Alias map is defense-in-depth.

### 5. Locked Decision #3 (Claude Agent SDK) formally revised
- Wave 5 abandoned Claude Agent SDK (can't run in Lambda — spawns `claude` CLI subprocess)
- Replaced with `@anthropic-ai/bedrock-sdk` `AnthropicBedrock` direct calls
- Documented in `PROJECT.md` Key Decisions table
- Phase 6 AGT-04 redesigned as explicit `@kos/context-loader::loadContext` helper
