    7. **Create `apps/dashboard/src/components/system/OfflineBanner.tsx`**:
       ```tsx
       'use client';
       import { useEffect, useState } from 'react';
       import { formatDistanceToNow } from 'date-fns';

       export function OfflineBanner() {
         const [online, setOnline] = useState(true);
         const [lastSync, setLastSync] = useState<Date | null>(null);
         useEffect(() => {
           setOnline(typeof navigator !== 'undefined' ? navigator.onLine : true);
           setLastSync(new Date());
           const onOn = () => { setOnline(true); setLastSync(new Date()); };
           const onOff = () => setOnline(false);
           window.addEventListener('online', onOn);
           window.addEventListener('offline', onOff);
           return () => { window.removeEventListener('online', onOn); window.removeEventListener('offline', onOff); };
         }, []);
         if (online) return null;
         const rel = lastSync ? formatDistanceToNow(lastSync, { addSuffix: true }) : 'never';
         return (
           <div role="status" className="fixed top-0 inset-x-0 z-40 bg-[var(--color-surface-2)] border-b border-[var(--color-border)] px-4 py-2 text-sm text-[var(--color-warning)] text-center">
             Offline · last synced {rel} · some actions disabled
           </div>
         );
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service worker→browser cache | Runs in isolated origin; single-user scope |
| SW cache→authenticated response | SW caches only 200 responses; redirects NEVER cached (P-02) |
| Offline banner→navigator.onLine | Pure client state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-12-01 | Tampering | SW cache poisoning | mitigate | cacheWillUpdate plugin rejects non-200 (P-02); Serwist hash manifest invalidates on every build. |
| T-3-12-02 | Information disclosure | Cached authenticated HTML | accept | Single-user device; cache lives only on Kevin's browser. Logout clears cookie → next SW-served HTML fails auth on API layer → user redirected to /login. |
| T-3-12-03 | DoS | Service worker loop | mitigate | `reloadOnOnline: true` + skipWaiting/clientsClaim keep updates clean; Serwist manages lifecycle. |
</threat_model>
