  <action>
    Build a Node CLI script (ESM `.mjs`) that exercises the full Phase 6 chain and asserts side-effects.

    **Modes:**
    - `--mock` (default when no AWS_REGION env): runs entirely offline using @kos/test-fixtures + mocked clients; verifies the SHAPE of the data flow (every contract parses, every Lambda's exported handler can be invoked with a mock event without throwing, every cross-package import resolves).
    - `--live` (requires AWS + Azure + Notion creds): emits a real EventBridge event, waits for downstream effects, queries RDS for assertions.

    **Mock-mode flow:**
    1. Import `fakeGranolaTranscript` from `@kos/test-fixtures`; build a `TranscriptAvailable` detail.
    2. Validate against `TranscriptAvailableSchema` (Zod). Assert valid.
    3. Import `services/transcript-extractor/src/handler.ts` `handler` and invoke with `{ 'detail-type': 'transcript.available', detail: <fixture> }` while monkey-patching:
       - `process.env.KEVIN_OWNER_ID` to a fake UUID.
       - `services/transcript-extractor/src/persist.ts::getPool` to return a mock pool.
       - `services/transcript-extractor/src/agent.ts::runExtractorAgent` to return a fake `Extract` with 2 action_items + 3 entity_mentions.
       - `services/transcript-extractor/src/notion.ts::writeActionItemsToCommandCenter` to return ['notion-page-1', 'notion-page-2'].
       - `services/transcript-extractor/src/persist.ts::publishMentionsDetected` to record calls (assert 3).
       - `services/transcript-extractor/src/persist.ts::insertTranscriptIndexed` to record call (assert 1).
    4. Assert no throw + correct call counts.
    5. Import `@kos/context-loader::loadContext`; call with `{ entityIds: ['00000000-0000-4000-8000-000000000001'], agentName: 'mock', captureId: 'mock-ulid', ownerId: 'mock-owner', pool: <mock pool> }`. Assert ContextBundle shape returned + assembledMarkdown is non-empty.
    6. Print summary table: each step PASS / FAIL.

    **Live-mode flow:**
    1. Validate AWS_REGION + KEVIN_OWNER_ID + at-least-one-test-transcript-id env present.
    2. PutEvents `transcript.available` to kos.capture with the test transcript_id.
    3. Poll for up to 60s:
       - SELECT FROM agent_runs WHERE capture_id = test_transcript_id AND agent_name = 'transcript-extractor' AND status = 'ok' — assert exists.
       - SELECT FROM transcripts_indexed WHERE transcript_id = test_transcript_id — assert exists.
       - SELECT FROM mention_events WHERE capture_id = test_transcript_id — assert >= 1.
    4. Trigger entity_dossiers_cached invalidation: insert mock mention_event for an entity with a cached dossier; SELECT FROM entity_dossiers_cached WHERE entity_id = mock — assert deleted by trigger.
    5. Print PASS/FAIL summary.

    **Exit codes:** 0 = all PASS; 1 = any FAIL.

    **Output format** (mock mode example):
    ```
    Phase 6 E2E (mock mode)
    [PASS] TranscriptAvailableSchema parses fakeGranolaTranscript
    [PASS] transcript-extractor handler invokes without throw
    [PASS] runExtractorAgent called once
    [PASS] writeActionItemsToCommandCenter received 2 action items
    [PASS] publishMentionsDetected called 3 times
    [PASS] insertTranscriptIndexed called once
    [PASS] @kos/context-loader::loadContext returns valid ContextBundle
    Result: PASS (7/7)
    ```

    Mirror the format conventions of `scripts/verify-phase-2-e2e.mjs` (existing — read first to copy idiom).

    - Zero hard dependency on AWS/Azure/GCP creds in mock mode.
    - Live-mode polling timeout 60 s.
    - Auto-detect mode via `--mock` flag OR absence of AWS_REGION env.
