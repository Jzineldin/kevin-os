<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Consumer Lambda → @kos/azure-search → Azure REST | hybridQuery sends BM25 + vector queries to Azure AI Search and returns matched documents. Document content originates from `entity_index`, `project_index`, and `agent_runs.output_json` — already-indexed user data, not external untrusted input. |
| Consumer Lambda → AWS Bedrock (Cohere v4 EU) | embedText sends rawText (mention text or transcript snippet) to Cohere for embedding. Same trust posture as the existing entity-resolver embedBatch path. |
| Consumer Lambda → AWS Secrets Manager | Lambda reads AZURE_SEARCH_ADMIN_SECRET_ARN to obtain Azure Search admin key + endpoint. Already used by 4 indexer Lambdas — extending to 4 agent Lambdas follows the same posture. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-AGT04-01 | I (Information disclosure) | IAM policy on agent Lambda → Azure Search admin secret | mitigate | Use `secret.grantRead(fn)` (CDK helper) which scopes secretsmanager:GetSecretValue to the exact secret ARN — NOT `Resource: '*'`. Mirrors the indexer Lambda IAM pattern. |
| T-06-AGT04-02 | E (Elevation of privilege) | bedrock:InvokeModel on Cohere v4 | mitigate | Resource ARN scoped to `arn:aws:bedrock:*:*:inference-profile/eu.cohere.embed-v4*` AND `arn:aws:bedrock:*::foundation-model/cohere.embed-v4*` only. NOT `Resource: '*'`. Copies the entity-resolver pattern verbatim. |
| T-06-AGT04-03 | T (Tampering) | OData filter built from entityIds in hybridQuery | accept | Existing IN-01 finding (out of scope per gap_closure_scope; covered by Plan 06-08). entityIds come from validated DB UUIDs (entity_index PK) at all 4 callsites — injection risk is theoretical, not practical. |
| T-06-AGT04-04 | D (Denial of service) | Cohere embedText call inside hybridQuery on every loadContext invocation | accept | Already gated by loadContext's Promise.all + .catch fallback — Azure failure surfaces as `partial: true` and degrades to empty semanticChunks. No new failure mode introduced. |
