    **`scripts/migrate-brain-dbs.mjs`** (~250 lines):

    Flow per DB:
    1. **Pre-flight:** Read `.notion-brain-dbs.json` (real, gitignored). Fail fast if any `id` starts with `REPLACE_`.
    2. **Retrieve current state:** `notion.databases.retrieve({ database_id })`. Capture: current title, current archived state.
    3. **Idempotency check:** If `archived === true` AND title already starts with `[MIGRERAD-`, skip with `[SKIP] already archived`. Don't re-write event_log.
    4. **Build new title:** `[MIGRERAD-${new Date().toISOString().slice(0,10)}] ${currentTitle}`.
    5. **Write event_log audit row (D-12):**
       ```sql
       INSERT INTO event_log (owner_id, kind, details, actor)
       VALUES ('kevin', 'brain-db-archived', $$({
         "database_id": "...",
         "original_title": "...",
         "new_title": "...",
         "archived_before": false,
         "reversibility_note": "restore via notion.databases.update(archived=false) within 90 days OR manual Notion trash recovery"
       })$$::jsonb, 'scripts/migrate-brain-dbs.mjs')
       RETURNING id;
       ```
    6. **Notion mutation:**
       ```typescript
       await notion.databases.update({
         database_id,
         archived: true,
         title: [{ type: 'text', text: { content: newTitle } }],
       });
       ```
    7. **Confirmation write-back to event_log:** UPDATE event_log SET details = details || '{"notion_ack_at": "ISO"}' WHERE id = <the-id>.
    8. On Notion API failure: event_log audit row already exists (marks INTENT); print the row id + retry instructions. Do NOT rollback the audit row — that's the point of write-ahead.

    **CLI flags:**
    - `--dry-run`: skip event_log INSERT + Notion PATCH; print proposed changes.
    - `--force`: re-archive even if already archived (overwrite title prefix with new date).
    - `--db-id <uuid>`: archive only this one (for targeted retries).

    **Retry policy:** Notion API 429 → 3 retries with exponential backoff (1s, 5s, 15s).

    **Output:** per-DB status lines + summary table at end. Exit 0 on all success/skip, 1 on any error.

    **Env required:** `NOTION_TOKEN`, `RDS_URL`. Read Notion token from AWS Secrets Manager if `NOTION_TOKEN` not in env (`kos/notion-integration-token`).

    **Post-archive note in output:** "Kevin must manually enable 'Lock database' in Notion UI for each archived DB — API does not expose this setting (see 10-RESEARCH.md §2)."
  </action>
