The package exports FOUR entrypoints:

```ts
// @serwist/turbopack — server-side (next.config.ts + Route Handler)
export declare const withSerwist: (nextConfig?: NextConfig) => NextConfig;
//     ^ BARE wrapper. Takes NO Serwist options. All options move to createSerwistRoute.
//       Internally adds ['esbuild', 'esbuild-wasm'] to nextConfig.serverExternalPackages.

export declare const createSerwistRoute: (options: InjectManifestOptions) => {
  dynamic: "force-static";
  dynamicParams: false;
  revalidate: false;
  generateStaticParams: () => Promise<{ path: string }[]>;
  GET: (_: Request, ctx: { params: Promise<{ path: string }> }) => Promise<NextResponse>;
};

// InjectManifestOptions (the shape passed to createSerwistRoute):
//   Required:   swSrc
//   Optional:   globPatterns, globDirectory, globIgnores, globStrict, globFollow,
//               additionalPrecacheEntries, dontCacheBustURLsMatching, manifestTransforms,
//               maximumFileSizeToCacheInBytes, modifyURLPrefix, templatedURLs, injectionPoint,
//               cwd, useNativeEsbuild, rebuildOnChange, esbuildOptions
//
//   NOTE: `swDest` DOES NOT EXIST in Turbopack. The route handler streams
//         the esbuild-bundled bytes directly; nothing is written to disk.
//         `/sw.js` is NOT emitted to `public/`. Static-generation paths
//         (set by generateStaticParams) live under the route segment that
//         contains the handler.
```

export interface SerwistProviderProps {
  swUrl: string;

  register?: boolean;

Key route-handler behaviours extracted from dist/index.js:

- `dynamic: "force-static"` + `generateStaticParams()` returns the set of
  file paths esbuild produced (sw.js, sw.js.map, and any chunk files) — so
  at `next build` time Next prerenders them as static assets. No runtime
  bundling on Vercel in prod.
- The GET handler sets `Content-Type: application/javascript` for `.js`
  and `application/json; charset=UTF-8` for `.map`, plus
  `Service-Worker-Allowed: /` so SW scope is root regardless of the route
  path.
- In dev (`NODE_ENV === 'development'`) the route handler rebuilds on
  every request if the SW source sha changes AND
  `additionalPrecacheEntries` is emptied AND `disablePrecacheManifest` is
  forced true. This is the new "dev mode" story — you don't disable via
  config, you use `SerwistProvider.disable` on the client plus the
  handler's built-in dev behaviour on the server.
- `useNativeEsbuild` defaults true on non-Windows. Vercel build runs on
  Linux, so we set `useNativeEsbuild: true` explicitly (belt-and-braces —
  removes the dependency on `esbuild-wasm` at build time).
