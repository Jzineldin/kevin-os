Promise

Kevin never has to re-explain context. When he mentions Damien, Christina, Almi, or any entity in his world, the system has already loaded the full dossier — every email, every meeting, every doc, every decision involving that entity — before responding. Every input gets routed automatically. Every output is context-aware. The system maintains itself.

5 hard gates

Quality gates between phases

Each gate is a list of must-be-true statements. No phase ships unless its gate passes. Gate 4 is a hard block — v2 agents cannot exist as code until v1 is daily-driven for 4 weeks.

GATE 1 Infrastructure ready (Phase 1 → 2)
  • CDK deploys clean: VPC, RDS Postgres 16+pgvector, 5 EventBridge buses, S3 with VPC Gateway Endpoint (zero NAT charges)
  • Notion Entities + Projects DBs with full schema; notion-indexer Lambda upserts to Postgres
  • Azure AI Search index created with binary quantization at creation time (92.5% cost reduction)
  • Custom vocabulary file deployed for AWS Transcribe (entity names, company names, Swedish + English tech terms)
  • Safety rails: cost alarms at $50/$100, archive-not-delete enforced, notification cap (3 Telegram/day) enforced at infra level, VPS scripts frozen
GATE 2 Minimum loop working (Phase 2 → 3)
  • Swedish ASR WER < 10% on 20 real Kevin voice samples (else escalate to KB-Whisper Fargate fallback)
  • Code-switched voice memo end-to-end < 25s, known entity matched, unknown queued in Inbox (no silent auto-merge)
  • Three-stage entity resolver working with audit trail on every merge
  • 50+ candidate dossiers from bulk import waiting in Inbox for batch review
  • DLQs on every EventBridge → Lambda rule; idempotency keys preventing double-process
GATE 3 Email triage safe to ship (Phase 4 internal)
  • Same email processed twice → exactly one draft created (idempotency)
  • Adversarial prompt-injection test email classified as data, no unauthorized action taken
  • EmailEngine runs 7 consecutive days without IMAP auth failure
  • Approve / Edit / Skip flow end-to-end — never sends without explicit input
HARD GATE 4 v2 agents unlock (Phase 8 → 9)
  • 4 continuous weeks of daily use (not reverted to VPS scripts or manual Notion)
  • Morning brief read and acted on at least 5 days/week
  • Entity resolver accuracy > 90% on voice inputs (measured over previous 2 weeks)
  • Email triage approval + edit rate > 70% (not skipped, over previous 2 weeks)
  • Dashboard opened > 3 sessions/week (Vercel analytics)
  • AGT-11 (investor-relations) additionally requires DPIA before deployment per EU AI Act
GATE 5 Baileys WhatsApp production label (within Phase 5)
  • 7 consecutive days with zero write operations in logs
  • Session auth in RDS — kill Fargate task, verify reconnection without QR scan
  • 4-hour minimum backoff on any session rejection verified
  • Zero mutation method calls in any Baileys log line

Locked architectural decisions

Cross-document research synthesis. These are decisions that shape everything downstream.

🔒
EventBridge-only event routing — n8n removed entirely
EventBridge + Lambda do in 20 lines what n8n needs 300+ lines of visual workflow for. Current VPS n8n (port 5678, no auth) is a live security risk. Frozen Phase 1, decommissioned Phase 10.
🔒
Claude Agent SDK + Bedrock, not LangGraph / CrewAI
Subagents as markdown files in .claude/agents/. Bedrock-native via CLAUDE_CODE_USE_BEDROCK=1. LangGraph adds 2-4 weeks of graph code for problems we don't have.
🔒
RDS PostgreSQL 16 (db.t4g.medium) — NOT Aurora Serverless v2
17% cheaper, no HNSW pgvector degradation. Reserved 1-year ~$31/mo. Source of truth lives in Notion; Postgres is a rebuildable index.
🔒
Notion = source of truth, Postgres = derived index
Agents write Notion first → confirm event → Postgres upsert. Human edits in Notion UI are canonical. Postgres rebuildable from Notion + S3.
🔒
Telegram = mobile push, Dashboard = desktop primary
iOS 17.4 removed standalone PWA push in EU under DMA — cannot engineer around. All mobile push goes through Telegram. Web Push is desktop-only.
🔒
SSE via Postgres LISTEN/NOTIFY — not AppSync, not Pusher
Single-user dashboard. SSE from Next.js Edge backed by Postgres NOTIFY. Native, free, instant.
🔒
Static Bearer token in Secrets Manager — no Cognito, no Clerk
Single user. 256-bit random token. Auth infrastructure is over-engineering at this scale.
🔒
Custom Chrome extension for LinkedIn — NOT Unipile
Runs in your normal browser session. Ban-safe. Unipile costs €49/mo + holds your cookie + LinkedIn Q1 2026 ban escalation made the third-party path lethal (HeyReach company page deleted).
Deliberate exclusions

What KOS will never do

As important as what it does. These are explicitly out of scope.

✕
Auto-send emails or auto-publish content
SES sends only after explicit Approve tap. Postiz schedules only after explicit Approve tap. Approval queue is a safety gate, not a bottleneck.
✕
Auto-accept calendar invites or book on your behalf
Reads, never writes. Calendar booking always requires explicit confirmation.
✕
Auto-categorization on capture
Categorization friction is the #1 reason second-brain systems die for ADHD users. KOS captures raw, classifies async via triage. You never see a "tag this" prompt.
✕
More than 3 Telegram pushes per day
Hard cap, enforced at infrastructure level in Phase 1. Morning brief + urgent-only escalation + day close. Everything else batches into Dashboard Inbox. Notification fatigue kills these systems at day 60.
✕
LinkedIn outbound automation
Read-only via Chrome extension. No auto-connect, no auto-DM. 23% ban rate on LinkedIn outbound automation in 2025-2026.
✕
v2 specialty agents before v1 has 4 weeks of daily use
Hard gate. Specifications can be written; code cannot ship until all 5 acceptance criteria are simultaneously met. Building specialists before the core works is the #2 abandonment pattern.
