Motion principles

All transitions use cubic-bezier(0.16, 1, 0.3, 1). Clean ease-out, no bounce.
Durations: 120ms for hover/focus, 180ms for state changes, 280ms for entry. Never over 280ms.
Hover changes border-color + background only. Never transform. No movement on hover.
Entry animations: opacity + translateY(6px) max. Never slide more than 8px.
Loading states: subtle pulse on a single 6×6 dot. Never spinners. Never skeleton screens.
Real-time updates: fade-in only. New row in a list slides 4px max. No layout reflow on existing rows.
Click feedback: brief scale(0.98) on buttons. 80ms in, 80ms out.
Sidebar nav active state: instant. No transition. Decisions feel snappy.

What this UI deliberately avoids

No glassmorphism, no neumorphism, no gradients on cards (only on the brand mark and one ai-block accent).
No drop shadows on UI elements. Only on the app frame itself for depth from page background.
No hover-grow, no card-tilt, no parallax. The system feels stable, not playful.
No bright neon accents. Single muted violet accent. Status colours desaturated (#34D399 not #00FF88).
No emoji-as-icon overload. Used only where they earn their place (Spår tags, status indicators).
No skeleton loaders. Empty states are honest text + a single dot. Loading is fast enough not to need them.
No bouncy springs, no elastic eases, no swirl animations. Cron-tier sobriety.
